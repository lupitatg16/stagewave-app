StageWave

Especificación de Requisitos del Sistema

Documento base para diseño de casos de prueba

StageWave es una aplicación web que permite comprar boletos para conciertos. El usuario selecciona el evento, la zona, la cantidad de boletos, el tipo de comprador, su edad, el método de pago y, opcionalmente, un código de descuento.

El sistema debe calcular el total, validar las reglas de compra y confirmar la orden si todos los datos son válidos. Ante cualquier error, se debe mostrar un mensaje descriptivo y no procesar la compra.

Interfaz de la Aplicación (Mockup)

El formulario tiene 6 secciones. Cada campo muestra su mensaje de error directamente debajo cuando la validación falla. El resultado aparece al final de la página al enviar el formulario.

8 Cálculo del Total
  • Subtotal = Precio de zona × Cantidad de boletos.
  • Después de descuento por tipo = Subtotal × (1 − % tipo comprador).
  • Total final = Monto anterior × (1 − % código promo), si aplica código.
  • Si no hay descuentos, el total es igual al subtotal.

Ejemplo: 2 boletos · Zona Preferente ($1,500) · Estudiante (10%) · STAGE10 (10%)
Subtotal = $3,000  →  −10% = $2,700  →  −10% = $2,430

9 Confirmación de Orden
  • Si todos los datos son válidos, el sistema muestra el resumen completo de la orden y un número de confirmación único.
  • El resumen debe mostrar: evento, fecha, zona, cantidad, tipo de comprador, método de pago, subtotal, descuentos aplicados y total final.
  • Si existe algún error, el sistema debe mostrar mensajes claros para cada campo incorrecto y no procesar la compra.
